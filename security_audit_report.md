# Security Audit Report (Deep, Brutal)

**Scope**: Full codebase scan for crypto misuse, insecure auth, injection risks, hard‑coded secrets.

## Key Findings

1. **Credential Exposure** – `foresight_mcp/auth.py` prints a generated readonly user password to stdout (line ~398). This leaks secrets in logs.
2. **Weak Password Hashing** – Passwords are hashed using a custom PBKDF2 implementation (`hashlib.pbkdf2_hmac`) with a comment "In production, use bcrypt or argon2". PBKDF2 is acceptable but lacks built‑in salting and stretch parameters; stronger Argon2/bcrypt is recommended.
3. **SQL Injection Surface** – Multiple modules (`temporal_queries.py`, `hybrid_retriever.py`, etc.) build SQL statements with f‑strings (`conn.execute(f"...{var}...", params)`). Though parameters are often passed, the surrounding f‑string can introduce injection if interpolated values contain SQL. Recommend using only parameter placeholders (`?`) and never embed raw variables.
4. **Mis‑use of Password Verification** – In `AuthManager._get_user_from_session` the verification call `self._verify_password("", password_hash)` checks only format, not the actual password. This is a logic flaw that could allow session bypass if hash parsing fails.
5. **Hard‑coded Default Admin Creation** – Admin password is generated and not persisted securely; although not logged, the creation flow should deliver credentials via a secure channel.
6. **Dynamic Execution** – No direct `eval/exec/os.system` usage in project code, but `yaml.load` and `pickle` are only present in third‑party packages, not in our source.

## Recommended Fixes

- **Remove Log of Read‑only Password**: Delete or guard the `print` statement; store credentials securely (e.g., env var, secret manager).
- **Upgrade Password Hashing**: Switch to `argon2-cffi` or `bcrypt` with appropriate work factor; deprecate the custom PBKDF2 helper.
- **Sanitize SQL Construction**: Refactor all f‑string SQL to use parameterized queries exclusively. Review `temporal_queries.py` and `hybrid_retriever.py` for any concatenated strings.
- **Correct Session Password Check**: Replace the placeholder verification with proper password validation against the stored hash.
- **Secure Admin Credential Delivery**: Output admin credentials through a secure channel (e.g., encrypted config file) and avoid printing.
- **Add Static Analysis**: Integrate security linters (bandit, sqlfluff) into CI to catch future issues.
